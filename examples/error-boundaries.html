Error Boundaries

Three levels of error handling: element-level, error boundaries, and global config.

1. Widget-Level Error Handling

Each widget has its own xh-error-template and xh-error-target. The error replaces just that widget's content. This is the same pattern used in the basic error handling example.

2. Error Boundary

A parent <div> with xh-error-boundary catches errors from child widgets that have no element-level error template. The boundary's error template renders into the boundary's xh-error-target container.

Boundary Handler

Widgets inside this boundary have no element-level error template. Errors walk up the DOM and are caught by this boundary.

Widget 1 - will succeed
Widget 2 - will fail 404
Widget 3 - will fail 500

3. Global Error Handler

When xhtmlx.config.defaultErrorTemplate and xhtmlx.config.defaultErrorTarget are set, any widget that has no element-level error template and no error boundary ancestor will use the global fallback.

4. Resolution Order Demo

This section demonstrates the full resolution chain. All three levels are active: global config is set, a boundary wraps the area, and one widget has its own element-level error template. The element-level handler wins over the boundary, which wins over the global.

Resolution order:
1. xh-error-template-{code} on the element itself
2. xh-error-template-{class} (e.g. 4xx) on the element
3. xh-error-template on the element
4. Nearest xh-error-boundary ancestor with matching template
5. xhtmlx.config.defaultErrorTemplate (global)
6. No template found: just adds xh-error CSS class

Error Handling Architecture

<!-- Level 1-3: Element-level -->
<button xh-get="/api/data"
        xh-error-template-404="/templates/not-found.html"
        xh-error-template="/templates/error.html"
        xh-error-target="#my-error">
  Load
</button>

<!-- Level 4: Error boundary -->
<div xh-error-boundary
     xh-error-template="/templates/boundary-error.html"
     xh-error-target="#boundary-errors">
  <button xh-get="/api/data">Load</button>
  <div id="boundary-errors"></div>
</div>

<!-- Level 5: Global config -->
<script>
  xhtmlx.config.defaultErrorTemplate = "/templates/error.html";
  xhtmlx.config.defaultErrorTarget = "#global-error";
</script>

Resolution chain:
  element xh-error-template-{code}
    → element xh-error-template-{class}
      → element xh-error-template
        → nearest xh-error-boundary
          → xhtmlx.config.defaultErrorTemplate
            → add xh-error CSS class
