Error template resolution order

<div xh-get="/api/data"
     xh-template="/templates/data.html"
     xh-error-template="/templates/error.html"
     xh-error-template-404="/templates/not-found.html"
     xh-error-template-422="/templates/validation.html"
     xh-error-target="#error-area">
</div>

Resolution order:
1. xh-error-template-{exact code}  (e.g. xh-error-template-404)
2. xh-error-template-{class}       (e.g. xh-error-template-4xx)
3. xh-error-template               (generic fallback)
4. No template: adds xh-error class, emits xh:responseError

Error data context:
{
  "status": 422,
  "statusText": "Unprocessable Entity",
  "body": { "error": "...", "fields": [...] }
}
